Instala OpenClaw en tu máquina local con un solo comando.
Sin tocar archivos. Sin conocimientos de Docker.

// PowerShell (como Administrador)
PS C:\> irm https://raw.githubusercontent.com/nexxoads/openclaw-easy-install/main/install.ps1 | iex
Requisito: Docker Desktop instalado y abierto.
WSL2: Si usas WSL2, abre Ubuntu/Debian y usa el comando de Linux.
// Terminal
$ curl -sSL https://raw.githubusercontent.com/nexxoads/openclaw-easy-install/main/install.sh | bash
Requisito: Docker Desktop para Mac instalado y en marcha.
// Terminal (Ubuntu / Debian / WSL2)
$ curl -sSL https://raw.githubusercontent.com/nexxoads/openclaw-easy-install/main/install.sh | bash
Sin Docker: curl -fsSL https://get.docker.com | sh
// ¿Qué hace el instalador?
01

Detecta tu entorno

Identifica macOS / Linux / WSL2 / Apple Silicon, verifica Docker instalado y en marcha.

02

Crea la carpeta de datos

Genera ~/openclaw-local/ con .env, docker-compose.yml y la carpeta data/.

03

Genera token de seguridad

Crea automáticamente un token aleatorio con openssl rand -hex 32 y lo guarda en .env.

04

Descarga la imagen oficial

Hace docker pull ghcr.io/openclaw/openclaw:latest desde el GitHub Container Registry oficial.

05

Arranca y verifica

Lanza el contenedor con restart: unless-stopped y hace polling a /healthz hasta confirmar que responde.

🦞 OpenClaw — Guía Comunitaria

Guía completa para instalar, configurar y gestionar OpenClaw con Docker desde cero. Sin conocimientos previos necesarios.

¿Qué es OpenClaw? Tu asistente de IA personal que corre en tu propia máquina. Se conecta a WhatsApp, Telegram, Discord y más. 100% open source en github.com/openclaw/openclaw.
¿Qué necesitas?
RequisitoDetalle
Docker DesktopLa única dependencia. Instálalo y listo.
Sistema operativomacOS, Windows (WSL2) o Linux (Ubuntu/Debian)
RAM mínima2 GB libres recomendados
Espacio en disco~1 GB para la imagen Docker
Estructura de carpetas tras la instalación
ÁRBOL DE ARCHIVOS
~/openclaw-local/ ├── docker-compose.yml # configuración del contenedor ├── .env # token y variables de entorno └── data/ # tus datos (persistentes) ├── openclaw.json # configuración de OpenClaw └── workspace/ # archivos y proyectos de agentes
📋 Requisitos previos

Solo necesitas Docker Desktop. Elige tu sistema:

Docker Desktop para macOS

Descarga desde docs.docker.com. Hay versión para Apple Silicon (M1/M2/M3) e Intel.

VERIFICAR INSTALACIÓN
docker --version docker compose version
Ambos deben responder con versión. El icono 🐳 en la barra de menú debe estar quieto (no animándose).
Docker Desktop para Windows

Descarga desde docs.docker.com. Durante la instalación activa "Use WSL 2 instead of Hyper-V".

Recomendado: Usa WSL2 (Ubuntu) en lugar de PowerShell nativo para mejor compatibilidad.
Instalar WSL2 si no lo tienes
POWERSHELL (Administrador)
wsl --install

Reinicia el PC. Luego en Docker Desktop: Settings → Resources → WSL Integration → activa Ubuntu → Apply & Restart.

Docker en Linux / WSL2
INSTALAR DOCKER (Ubuntu/Debian)
curl -fsSL https://get.docker.com | sh sudo usermod -aG docker $USER newgrp docker
ARRANCAR DOCKER AL INICIO
sudo systemctl enable docker sudo systemctl start docker

🚀 Primer uso

Sigue estos pasos tras la instalación para configurar OpenClaw por primera vez.

1. Abre el panel de control
NAVEGADOR
http://127.0.0.1:18789
2. Obtén tu token de acceso
TERMINAL
cat ~/openclaw-local/.env

Copia el valor de OPENCLAW_GATEWAY_TOKEN y pégalo en el campo Shared Secret del panel de control.

3. Onboarding inicial (solo una vez)
TERMINAL
docker exec -it openclaw-local node dist/index.js onboard --mode local --no-install-daemon

Te pedirá tu API key del proveedor de IA (Anthropic, OpenAI, etc.) y configurará los canales de mensajería.

4. Verificar que funciona
TERMINAL
curl -fsS http://127.0.0.1:18789/healthz # liveness curl -fsS http://127.0.0.1:18789/readyz # readiness
Si ambos responden, OpenClaw está funcionando correctamente.
5. Conectar un canal de mensajería
TERMINAL
# Telegram docker exec -it openclaw-local node dist/index.js channels add --channel telegram --token "TU_TOKEN_BOT" # WhatsApp (escanea QR en terminal) docker exec -it openclaw-local node dist/index.js channels login # Discord docker exec -it openclaw-local node dist/index.js channels add --channel discord --token "TU_TOKEN"
🔄 Actualizar

Actualizar a la última versión no borra tus datos. Solo descarga la nueva imagen y reinicia.

Actualización en un comando
TERMINAL
cd ~/openclaw-local && docker compose pull && docker compose up -d
Tus datos están a salvo. El directorio data/ nunca se toca durante actualizaciones.
Ver qué versión tienes instalada
TERMINAL
docker inspect openclaw-local --format "{{.Config.Image}}"
Fijar una versión específica

Edita OPENCLAW_IMAGE en ~/openclaw-local/.env:

.env
OPENCLAW_IMAGE=ghcr.io/openclaw/openclaw:2026.2.26

Consulta todas las versiones en github.com/openclaw/openclaw/releases.

💻 Comandos Docker

Referencia completa de todos los comandos que necesitarás.

Estado y monitoreo
TERMINAL
docker ps # contenedores activos docker ps -a # todos (incluye parados) docker logs openclaw-local # últimos logs docker logs -f openclaw-local # logs en tiempo real docker logs --tail 50 openclaw-local # últimas 50 líneas docker stats openclaw-local # CPU y RAM en tiempo real
Arrancar / Parar / Reiniciar
TERMINAL
docker start openclaw-local # arrancar docker stop openclaw-local # parar (datos seguros) docker restart openclaw-local # reiniciar
Docker Compose (desde ~/openclaw-local)
TERMINAL
cd ~/openclaw-local docker compose up -d # arrancar en background docker compose down # parar (datos seguros) docker compose pull # descargar imagen más nueva docker compose ps # estado de servicios docker compose logs -f # logs de todos los servicios
Comandos OpenClaw via docker exec
TERMINAL
# Onboarding docker exec -it openclaw-local node dist/index.js onboard --mode local --no-install-daemon # Estado del gateway docker exec -it openclaw-local node dist/index.js gateway status # Listar canales configurados docker exec -it openclaw-local node dist/index.js channels list # Shell dentro del contenedor docker exec -it openclaw-local bash
⚙️ Configuración

Variables y archivos clave para personalizar tu instalación.

El archivo .env
~/openclaw-local/.env
# Imagen (cambia la versión para fijar una específica) OPENCLAW_IMAGE=ghcr.io/openclaw/openclaw:latest # Token de acceso al panel (¡no lo compartas!) OPENCLAW_GATEWAY_TOKEN=tu_token_secreto # Rutas de datos (no cambiar salvo que sepas lo que haces) OPENCLAW_CONFIG_DIR=/Users/tuusuario/openclaw-local/data OPENCLAW_WORKSPACE_DIR=/Users/tuusuario/openclaw-local/data/workspace # Zona horaria OPENCLAW_TZ=Europe/Madrid
Cambiar el puerto

Si el puerto 18789 está ocupado, edita docker-compose.yml:

docker-compose.yml
ports: - "18790:18789" # puerto_en_tu_maquina:puerto_del_contenedor

Luego reinicia: cd ~/openclaw-local && docker compose up -d

🗑️ Desinstalar

Elimina OpenClaw completamente de tu sistema.

Aviso: Esto elimina el contenedor, la imagen y todos tus datos en data/. Haz copia de seguridad si los necesitas.
Desinstalador automático
TERMINAL
curl -sSL https://raw.githubusercontent.com/nexxoads/openclaw-easy-install/main/uninstall.sh | bash

Pedirá confirmación antes de borrar nada.

POWERSHELL
irm https://raw.githubusercontent.com/nexxoads/openclaw-easy-install/main/uninstall.ps1 | iex
Paso 1 — Parar y eliminar el contenedor
TERMINAL
docker stop openclaw-local docker rm openclaw-local
Paso 2 — Eliminar la imagen Docker
TERMINAL
docker rmi ghcr.io/openclaw/openclaw:latest
Paso 3 — Eliminar archivos
macOS / Linux
rm -rf ~/openclaw-local
Windows PowerShell
Remove-Item -Recurse -Force "$env:USERPROFILE\openclaw-local"
🐛 Errores comunes

Soluciones a los problemas más frecuentes.

❌ pull access denied / repository does not exist
ERROR
Error response from daemon: pull access denied for phioranex/openclaw-docker
Causa: Tienes un docker-compose.yml antiguo con una imagen incorrecta.

Solución: Desinstala y reinstala limpio:

TERMINAL
docker stop openclaw-local 2>/dev/null; docker rm openclaw-local 2>/dev/null rm -rf ~/openclaw-local curl -sSL https://raw.githubusercontent.com/nexxoads/openclaw-easy-install/main/install.sh | bash
❌ version is obsolete (warning)
WARNING
WARN: the attribute `version` is obsolete, it will be ignored
No es un error crítico, solo un aviso. El docker-compose.yml antiguo tiene version: '3.8' que ya no es necesario en Compose v2. Reinstala con el script actualizado para eliminarlo.
❌ Docker no está arrancado
ERROR
Cannot connect to the Docker daemon. Is the docker daemon running?
SistemaSolución
macOS / WindowsAbre Docker Desktop y espera al icono de la ballena quieto
Linuxsudo systemctl start docker
WSL2Abre Docker Desktop en Windows y activa la integración WSL2
❌ Puerto 18789 ya en uso
ERROR
Bind for 0.0.0.0:18789 failed: port is already allocated
docker-compose.yml — cambiar puerto
ports: - "18790:18789" # cambia 18790 por el puerto libre que prefieras
TERMINAL
cd ~/openclaw-local && docker compose up -d
❌ Contenedor arranca y se cae
TERMINAL
docker logs openclaw-local # ver el error exacto docker ps -a # ver estado (Exited, Restarting...)
❌ Permission denied (Linux)
TERMINAL
sudo usermod -aG docker $USER newgrp docker
♻️ Reinstalación limpia

Borra absolutamente todo y empieza desde cero.

Cuidado: Elimina todos tus datos de OpenClaw. Solo hazlo si realmente necesitas empezar de cero.
macOS / Linux / WSL2
TERMINAL — ejecutar todo de una vez
# 1. Parar y eliminar contenedor docker stop openclaw-local 2>/dev/null || true docker rm openclaw-local 2>/dev/null || true # 2. Eliminar imagen (fuerza re-descarga completa) docker rmi ghcr.io/openclaw/openclaw:latest 2>/dev/null || true # 3. Eliminar todos los archivos y datos rm -rf ~/openclaw-local # 4. Instalar de nuevo desde cero curl -sSL https://raw.githubusercontent.com/nexxoads/openclaw-easy-install/main/install.sh | bash
Windows (PowerShell)
POWERSHELL
docker stop openclaw-local; docker rm openclaw-local docker rmi ghcr.io/openclaw/openclaw:latest Remove-Item -Recurse -Force "$env:USERPROFILE\openclaw-local" irm https://raw.githubusercontent.com/nexxoads/openclaw-easy-install/main/install.ps1 | iex
Limpiar caché de Docker (opcional)
TERMINAL
docker system prune -f # elimina imágenes, redes y caché no usados
No borra contenedores ni volúmenes activos. Solo limpia recursos huérfanos.
